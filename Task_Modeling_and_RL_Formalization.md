# 强化学习与复杂推理任务建模

在利用强化学习（RL）提升小型语言模型（SLM）的逻辑推理能力时，核心挑战在于将生成任务转化为马尔可夫决策过程（MDP），并为后续的统计论证提供可观测量。本章详细阐述多步算术表达式构造任务的定义、建模方法以及复合奖励结构。

## 3.1 多步算术表达式构造任务定义

Arithmetic-24 任务要求模型在 `<think>` 标签内进行多步推演，并在其后给出唯一的算术表达式。该任务将复杂的内部搜索树平摊为自回归生成的词块序列，为后续针对逻辑密度与成功率的相关性分析（见第五章）提供了数据切片基础。

## 3.2 MDP 形式化建模

我们将文本生成建模为离散时间 MDP：
- **状态空间 $\mathcal{S}$**：当前 Prompt 及已生成的 Token 序列。
- **动作空间 $\mathcal{A}$**：模型的全词表。
- **状态转移**：由于语言模型的自回归特性，状态转移是确定性的。
- **策略 $\pi_\theta$**：智能体在给定上下文下预测下一个词的概率分布。

## 3.3 复合多阶段奖励函数设计

为解决强化学习中极其稀疏的奖励信号问题，并引导模型生成规范的推理过程，本实验摒弃了简单的二元奖励机制，设计了**复合多阶段过滤与惩罚奖励结构（Composite Multi-stage Reward）**。该函数由多个依次递进的检验阶段构成：
- **第一阶段：严格格式与废话惩罚**。在输出结果中，强制验证模型是否仅输出了独一且正确的 `<think>` 与 `</think>` 标签。若检测到多个闭合标签或包含了任何形式的解说性中英文本，将直接予以最大力度的负向惩罚（-1.0）并提前截断，从而在早期杜绝无效生成的探索震荡。
- **第二阶段：过程监督与幻觉监控（PRM）**。针对中间推理过程（CoT），系统将提取类似 `A = B` 的形式逻辑步，依靠极速且安全的 `eval()` 二次评估左侧的计算结果是否等于右侧。对于查出造假的等式进行严厉的"作弊扣分"（-1.0）；对合法的运算推演则按正确步数给予定向奖励，作为逻辑密度的底层牵引。
- **第三阶段：表达式校验与终态评判**。在最终公式阶段，通过多重集机制严格审查生成的表达式是否且仅使用了要求的初始数字一次。随后验证是否包含多个等号缩串计算或指数违规，最终基于计算真值：精确等于 24 获得主成功牵引的正向高分（+4.0）；一切非本原解空间的行为（如数值偏差、语法报错等）将得到轻重不同的分类扣分。

## 3.4 行为约束与组内优势方差性质

对于 GRPO 算法，本章重点探讨其**组内相对优势（Group-based Advantage）**的方差性质。
根据经典的理论推断，当组规模 $G$ 增加时，组内基准（Baseline）的估计方差应与 $1/G$ 成比例缩减。这一理论假设将直接作为第五章消融实验中 ANOVA 分析的核心探讨点。值得一提的是，在我们的真实实验数据中，由于受限的采样多样性以及序列长度变化，反而观察到了在特定算力约束下大 $G$ 值（如 $G=16$ 时）导致组内优势标准差（adv_std）不降反升的反直觉统计结果（详见第 5 章方差齐性检验分析）。
通过 3.3 节所述的强制行为约束机制（PRM 指导），我们确保模型在探索空间的搜索分枝能在第一时刻被集中拉回到逻辑自洽的路径上。

## 3.5 奖励稀疏性与探索难度分析

通过 SFT 预热和逻辑加权的 Dense 奖励，我们将"大海捞针"式的随机探索转化为有方向的统计归导。这种建模方式不仅缩短了模型达到收敛阈值的时间，更为后续对两种算法在"单位时间效率比"上的对比统计奠定了理论基础。

## 3.6 长度感知优势归一化（Length-Aware Advantage Normalization）

在标准 GRPO 中，每条响应获得一个标量优势 $\hat{A}_i$，该优势在策略梯度计算时被均匀广播到该响应的每个 Token 上。设响应 $i$ 的生成长度为 $L_i$，则其对策略梯度的**总贡献**与 $L_i \cdot \hat{A}_i$ 成正比。这意味着在奖励信号相同的前提下，更长的响应天然获得了更多的梯度更新量，从而引入了一种系统性的**长度偏差（Length Bias）**——模型被隐式激励去生成更长的序列以最大化其对参数的影响力，而非追求推理的准确性。

## 3.7 平滑奖励退火（Smooth Reward Annealing）

本文提出基于 Sigmoid 的**平滑退火机制**，将 Dense 奖励 $r^{\text{dense}}$ 与 Binary 奖励 $r^{\text{binary}}$ 按可微权重进行连续插值：

$$r = (1 - \alpha) \cdot r^{\text{dense}} + \alpha \cdot r^{\text{binary}}$$

其中退火系数 $\alpha$ 由 EMA 成功率 $\bar{s}$ 通过 Sigmoid 函数自动调节。

## 3.8 优势裁剪（Advantage Clipping）

在 24 点任务中，奖励分布呈现显著的双峰结构。本文引入**优势裁剪**，将归一化后的优势值限制在特定区间内：

$$\hat{A}_i^{\text{clip}} = \text{clamp}(\hat{A}_i, -c, +c), \quad c = 3.0$$

## 3.9 可解性预过滤（Solvable-Only Curriculum）

本文利用递归穷举求解器，在训练前对数据集进行**可解性预判**，过滤掉所有无解题目。这一改进对规则型可验证环境（如数学推理、代码生成等具有明确正确性判据的任务）具有普适意义。

## 3.10 组内多样性奖励（Intra-group Diversity Bonus）

在强化学习的探索阶段，模型往往会陷入“模式坍塌（Mode Collapse）”，即针对同一题目生成的 $G$ 个样本在逻辑路径上极度趋同。为了强制模型探索解空间的不同分枝，本文在优势计算后引入**多样性奖金因子**：

$$\hat{A}_i^{\text{div}} = \hat{A}_i + \lambda \cdot \frac{\text{Count}(\text{Unique Patterns in Group})}{G}$$

其中 $\text{Unique Patterns}$ 通过提取响应中的运算符指纹（Operator Fingerprints）进行识别。该机制鼓励模型在同一个 Batch 内尝试不同的算术思路，从而在统计上提高了发现正确解的概率。
